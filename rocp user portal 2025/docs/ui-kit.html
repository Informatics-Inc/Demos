Badge vs Pill

Badges and pills use the same base component (.badge) but communicate different intent. Use them consistently so scanning tables and dashboards feels predictable.

Badges

Use a badge to communicate status/state. Badges answer: “What state is this in?” They’re ideal for tables and workflow indicators.

Approved Expiring Overdue New

Pill

Use a pill for categorization/attributes. Pills answer: “What is this related to?” They’re often grouped and read like tags.
